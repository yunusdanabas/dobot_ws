\section{Overview}
% ──────────────────────────────────────────────────────────────────────────────

This lab session focuses on setting up your development environment and verifying the connection with the Dobot Magician robotic arm. You will learn how to interface with the robot using Python and perform basic joint-space control.

\subsection*{Included Scripts}

\renewcommand{\arraystretch}{1.5}
\begin{tabular}{@{}lp{12cm}@{}}
  \toprule
  \textbf{Script} & \textbf{Purpose and Usage} \\
  \midrule
  \texttt{01\_init\_check.py} & \textbf{Purpose:} Verifies the hardware connection, clears active alarms, and moves the robot to a safe ready position. \newline \textbf{Usage:} Run this first to ensure your setup is functional. \\
  \texttt{02\_joint\_control.py} & \textbf{Purpose:} Provides an interactive terminal interface (REPL) to move individual joints. \newline \textbf{Usage:} Enter joint angles in degrees to observe the robot's motion. \\
  \bottomrule
\end{tabular}

\section{Hardware Configuration}
% ──────────────────────────────────────────────────────────────────────────────

\subsection{Physical Setup Protocol}

\begin{enumerate}
  \item \textbf{Surface:} Place the robot on a flat, stable surface. Clear a 40\,cm radius around the base.
  \item \textbf{Posture:} Manually set the arm to a neutral posture (both segments at $\approx$45$^\circ$) before powering on to prevent sudden torque spikes.
  \item \textbf{Power:} Connect the 12V DC adapter. \textbf{USB power alone is insufficient} for motor operation.
  \item \textbf{Communication:} Connect the USB cable. The robot uses a CP210x USB-to-UART bridge.
\end{enumerate}

\subsection{Status Signals}

\begin{itemize}
  \item \textbf{Green LED + Single Beep:} System initialized and ready.
  \item \textbf{Red LED + Rapid Beeping:} Alarm state (usually a collision or limit violation). Run \texttt{01\_init\_check.py} to clear.
  \item \textbf{Yellow/Flashing LED:} Busy (homing or executing a command).
\end{itemize}

% ──────────────────────────────────────────────────────────────────────────────
\section{Software and OS Setup}
% ──────────────────────────────────────────────────────────────────────────────

\subsection{Linux Setup}
Users must have permission to access serial devices. Run the following once:
\begin{lstlisting}[style=BashStyle]
sudo usermod -a -G dialout $USER
# Log out and log back in for changes to take effect.
\end{lstlisting}
The robot typically appears as \texttt{/dev/ttyUSB0} or \texttt{/dev/ttyACM0}.

\subsection{Windows Setup}
The driver is usually handled by Windows Update. To find the port:
\begin{enumerate}
  \item Open \textbf{Device Manager}.
  \item Expand \textbf{Ports (COM \& LPT)}.
  \item Look for \textbf{Silicon Labs CP210x USB to UART Bridge} (e.g., \texttt{COM3}).
\end{enumerate}

\subsection{macOS Setup}
Modern macOS versions include the driver. The device will appear as:
\texttt{/dev/cu.usbserial-*} or \texttt{/dev/cu.SLAB\_USBtoUART}.

\section{Troubleshooting}
% ──────────────────────────────────────────────────────────────────────────────

\begin{tcolorbox}[infobox, title={Common Connection Issues}]
\rowcolors{2}{white}{ToneBlueBg}
\begin{tabular}{@{}p{5cm}p{10cm}@{}}
  \toprule
  \textbf{Symptom} & \textbf{Recommended Action} \\
  \midrule
  \texttt{Access Denied} or \texttt{Port Busy} & Ensure \textbf{DobotStudio} and all other Python scripts are closed. Only one process can own the serial port. \\
  \texttt{Device not found} & Verify both USB and Power cables. Check Device Manager (Win) or \texttt{ls /dev/tty*} (Unix). \\
  Motors don't move & Check the 12V power supply. If the light is red, run \texttt{01\_init\_check.py} to clear alarms. \\
  Erratic movement & Reset to neutral posture and power cycle the robot. \\
  \bottomrule
\end{tabular}
\end{tcolorbox}

\section{Safety Mandates}
% ──────────────────────────────────────────────────────────────────────────────

\begin{tcolorbox}[safetybox, title={Mandatory Safety Rules}]
\begin{enumerate}
  \item \textbf{Workspace:} Maintain a clean area. No cables or personal items in the reach zone.
  \item \textbf{Initialization:} Always run \texttt{01\_init\_check.py} before starting your own scripts.
  \item \textbf{Helper Functions:} Use \texttt{safe\_move()} from \texttt{utils.py}. It enforces software limits to prevent physical damage.
  \item \textbf{Supervision:} Stay within reach of the power cable for immediate emergency disconnection.
\end{enumerate}
\end{tcolorbox}
