\section{Hardware Setup}
% ──────────────────────────────────────────────────────────────────────────────

\begin{enumerate}
  \item \textbf{Surface:} Place on a flat, stable surface. Clear a \qty{40}{\cm} radius around the base.
  \item \textbf{Posture:} Manually set the arm to a neutral posture (both segments at $\approx$\qty{45}{\degree}) before powering on.
  \item \textbf{Power:} Connect the \qty{12}{\V} DC adapter. \textbf{USB power alone is insufficient.}
  \item \textbf{Communication:} Connect the USB cable (CP210x USB-to-UART bridge).
\end{enumerate}

\textbf{Status Signals:}
\begin{itemize}
  \item \textbf{Green LED + Single Beep:} Ready.
  \item \textbf{Red LED + Rapid Beeping:} Alarm state — click \textbf{Clear Alarms} in the GUI.
  \item \textbf{Yellow/Flashing LED:} Busy (homing or executing a command).
\end{itemize}

% ──────────────────────────────────────────────────────────────────────────────
\section{OS-Specific Notes}
% ──────────────────────────────────────────────────────────────────────────────

\subsection{Linux}
Users must have permission to access serial devices. Run the following once:
\begin{lstlisting}[style=BashStyle]
sudo usermod -a -G dialout $USER
# Log out and log back in for changes to take effect.
\end{lstlisting}
The Magician typically appears as \texttt{/dev/ttyUSB0} or \texttt{/dev/ttyACM0}.

\subsection{Windows}
The driver is usually handled by Windows Update. To find the port manually:
\begin{enumerate}
  \item Open \textbf{Device Manager}.
  \item Expand \textbf{Ports (COM \& LPT)}.
  \item Look for \textbf{Silicon Labs CP210x USB to UART Bridge} (e.g., \texttt{COM3}).
\end{enumerate}
The GUI auto-discovers the port, so manual lookup is typically unnecessary.


% ──────────────────────────────────────────────────────────────────────────────
\section{The Dobot Magician}
% ──────────────────────────────────────────────────────────────────────────────

The \textbf{Dobot Magician} is a 4-axis desktop robotic arm with:
\begin{itemize}
  \item \textbf{Reach:} $\approx$\qty{320}{\mm} maximum.
  \item \textbf{Payload:} \qty{500}{\g} at full extension.
  \item \textbf{Repeatability:} $\pm$\qty{0.2}{\mm}.
  \item \textbf{Communication:} USB-serial (CP210x chip).
\end{itemize}

The arm has two main segments connected by revolute joints:
\begin{itemize}
  \item \textbf{Upper arm (L1):} \qty{135}{\mm} — from the shoulder (J2) to the elbow.
  \item \textbf{Forearm (L2):} \qty{147}{\mm} — from the elbow to the end-effector.
\end{itemize}

A \textbf{parallel linkage} on J3 keeps the forearm's firmware angle relative to the upper arm (body-frame), rather than world-frame absolute.

% ──────────────────────────────────────────────────────────────────────────────
\section{Using the GUI}
% ──────────────────────────────────────────────────────────────────────────────

\begin{enumerate}
  \item \textbf{Launch:} \texttt{python 00\_magician\_gui.py}
  \item \textbf{Connect:} Select the port (auto-discovered with a star marker) and click \textbf{Connect}. The GUI clears any alarms and homes the robot automatically.
  \item \textbf{Move Joints:} Use the spinboxes and \textbf{+/$-$} buttons to set a target angle, then click \textbf{Move} for that joint, or \textbf{Send All} to move all four joints at once.
  \item \textbf{Live Pose:} The right panel shows the current X, Y, Z, R and J1--J4 values, updated every \qty{500}{\ms}.
  \item \textbf{Joint Limits:} The limits panel shows the safe range for each joint.
  \item \textbf{Details:} The log panel at the bottom shows connection events, alarms, and move commands.
\end{enumerate}

\section{Troubleshooting}
% ──────────────────────────────────────────────────────────────────────────────

\begin{tcolorbox}[infobox, title={Common Connection Issues}]
\rowcolors{2}{white}{ToneBlueBg}
\begin{tabularx}{\linewidth}{@{}lX@{}}
  \toprule
  \textbf{Symptom} & \textbf{Recommended Action} \\
  \midrule
  \texttt{Access Denied} & Ensure \textbf{DobotStudio} and other Python scripts are closed. \\
  \texttt{Device not found} & Verify both USB and Power cables. Click \textbf{Refresh} in the GUI to refresh ports. \\
  Motors don't move & Check the \qty{12}{\V} supply. If LED is red, click \textbf{Clear Alarms} in the GUI. \\
  Erratic movement & Reset to neutral posture and power cycle the robot. \\
  \bottomrule
\end{tabularx}
\end{tcolorbox}


% ──────────────────────────────────────────────────────────────────────────────
\section{Safety Mandates}
% ──────────────────────────────────────────────────────────────────────────────

\begin{tcolorbox}[safetybox, title={Mandatory Safety Rules}]
\begin{enumerate}
  \item \textbf{Workspace:} Maintain a clean area. No cables in the reach zone (\qty{40}{\cm} radius around the base).
  \item \textbf{Initialization:} Always let the GUI complete its connection sequence (alarm clear + homing) before sending commands.
  \item \textbf{Joint Limits:} The GUI clamps all values to the safe ranges shown in the Joint Limits panel.
  \item \textbf{Supervision:} Stay within reach of the power cable at all times.
\end{enumerate}
\end{tcolorbox}
